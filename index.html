  • ES5 in 2009
  • ES6 in 2015 (ES2015)

Object rest/spread properties 

// Rest
const obj = { foo: 1, bar: 2, baz: 3 };
const { foo, ...rest } = obj;
// foo is 1
// rest is { bar: 2, baz: 3 }

// Spread
const obj = { foo: 1, bar: 2, baz: 3 };
const other = { ...obj, qux: 4 };
// other is { foo: 1, bar: 2, baz: 3, qux: 4 }
  • Can be used instead of Object.assign to merge objects
  • Object.assign and spread have slightly different schematics

Named regex captures 

const fsPath = "/root/path/to/the/Resource";
const regexp = /\/
  (?<hierarchy>[^\/]+)
  (?<membership>\/.*?)[\/]*
  (?<name>[^\/]*)
  $/u;

const { groups } = regexp.exec(fsPath)
// {
//   hierarchy:  "root",
//   membership: "/path/to/the",
//   name:       "Resource",
// }
  • More readable and maintainable code
  • No need to change code if order of groups changes

Dynamic import() 

const main = document.querySelector("main");
const dynamicModule = "myfile";

import(`./dynamic/${ dynamicModule }.js`)
  .then(module => {
    module.loadPageInto(main);
  })
  .catch(err => {
    main.textContent = err.message;
  });
  • Current import modules are static declarations
  • Dynamically load parts of JS application at runtime
  • E.g. runtime values, performance, robustness

New Class features 

class Counter {
  // Public, private and static fields
  publicValue = 0;
  #privateValue = 0;
  static staticField = 0;

  // Private getters and setter
  get #value() { return #privateValue; }
  set #value(val) {
    this.#privateValue = val;
  }

  constructor() {}

  // Private methods
  #increment() {
    this.#privateValue++;
  }
}
  • 3 different proposals
    • Private methods
    • Class fields
    • Static class features

BigInt: Arbitrary precision integers

JS only supports integers up to 253-1

// Literal + constructor
const theBiggestInt = 9007199254740991n;
const alsoHuge = BigInt(9007199254740991);
const hugeButString = BigInt('9007199254740991');
// ↪ 9007199254740991n

// Same operators as with Number
const bigN = 2n ** 54n;
// ↪ 18014398509481984n
  • New primitive
  • Has same operators as Number
  • Already in Chrome, underway for TS

Array.prototype.{flatMap,flat} 

// Array.prototype.flat( [ depth ] )
const nestedArr = [[1], [2, 3], [4]]
nestedArr.flat()
// > [1, 2, 3, 4]

// Array.prototype.flatMap( mapperFunction [ , thisArg ] )
const nestedArr = [[1], [2, 3], [4]]
nestedArr.flatMap(x => x + 1)
// > [2, 3, 4, 5]
  • The smoosh gate

Object.fromEntries

Reverse of Object.entries 
obj = Object.fromEntries([
  ['a', 0],
  ['b', 1]
]);

// obj is { a: 0, b: 1 }

Decorators 

@defineElement("num-counter")
class Counter extends HTMLElement {
  @observed #x = 0;

  @bound
  #clicked() {
    this.#x++;
  }

  @bound
  render() {
    this.textContent = this.#x.toString();
  }
}
  • Allows adding behavior to an individual object
